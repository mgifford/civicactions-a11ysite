Contributing Guidelines

For All Contributions

  • Focus on practical guidance that helps people implement accessibility
  • Include real examples and case studies when possible
  • Cite authoritative sources like WCAG, Section 508, and accessibility experts
  • Write in plain language that's accessible to diverse audiences

Code Contributions

  • Follow accessibility best practices in any code contributions
  • Test with assistive technology when adding interactive features
  • Include accessibility considerations in pull request descriptions
  • Document accessibility decisions and rationale

Content Contributions

  • Verify links and resources are current and accurate
  • Include context about why resources are recommended
  • Consider diverse user needs in examples and recommendations
  • Maintain consistent voice and style with existing content
